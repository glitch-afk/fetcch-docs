# Proof of ID

**Provider Registry** needs **Proof of ID** to be able to create new IDs in their namespace.

**Proof of ID** is data in format

```json copy
{
  "id": "satyam@fetcch",
  "pid": "O11x4IdY-058Cl_7xkunC4LUKyWQdkE-NRM8BNQyDmE",
  "pidType": "ARWEAVE",
  "address": {
    "default": "0x16cCA730925E3DB8E60BBa9e564Aff4d588E0364",
    "others": ["0x3cB51236f17459f11fc5E934477762839caC5133", "0x16cCA730925E3DB8E60BBa9e564Aff4d588E0364"]
  },
  "merkelRoot": "59672cb5533759a1ecfd620cbb40be1a243522c056d5c67b7100adf1cf3162f7",
  "signature": "0x000000" // only if msg.sender isn't default.address
}
```

Anybody could verify IDs using `merkleRoot` present on the chain.
`signature` is needed when PID is updated by non `default.address` as `msg.sender`
