### Update Identity

>    ðŸ’¡ `secret-key` required

> Updating secondary addresses from previous identity


> ðŸ’¡ if `default.address` is being updated from APTOS to some chain, you must send `extraData` along with identity in format mentioned below, `extraData` will contain previous `publicKey` which will be used to verify `previousSignature` signed by previous `default.address`

```jsx
{
	...identity,
	"extraData": {
		"publicKey": "0x...."
	}
}
```

> ðŸ’¡ Above is only applicable if `default.chain` is updated from APTOS to some chain (incl APTOS)

Route - `/v1/identity/`
    
Method - `PATCH`

Request Body -

```json
{
  "identifier": "sax",
  "provider": "metamask",
  "default": {
    "address": "0x9aDF83d1bB317DA878C8677E5d108CecFcFe290c",
    "chain": 3,
    "isContract": false
  },
  "secondary": [],
  "currentSignature": "SIGNATURE"
}
```

Response - 

```json
{
  "status": 200,
  "timestamp": "",
  "data": {
    "id": "sax@metamask",
    "identifier": "sax",
    "provider": {...provider object},
    "default": {
      "address": "0x9aDF83d1bB317DA878C8677E5d108CecFcFe290c",
      "chain": {...chain object},
      "isContract": false
    },
    "secondary": [],
    "currentSignature": "SIGNATURE"
  }
}
```
